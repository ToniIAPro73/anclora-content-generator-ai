# Skill: Tenancy Hardening

Usa esta habilidad cuando una feature toque identidad, workspace resolution o aislamiento de datos.

## Checklist

1. Identificar endpoints que aceptan `workspaceId` del cliente.
2. Resolver workspace en server desde auth.
3. Limpiar fetches cliente que exponen IDs sensibles.
4. Documentar fallbacks temporales.
5. Cerrar con spec, test plan y notas de riesgos.
