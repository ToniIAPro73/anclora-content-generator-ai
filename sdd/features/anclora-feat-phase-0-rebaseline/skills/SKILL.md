# Skill: Phase 0 Rebaseline

Usa esta habilidad cuando haya que re-alinear documentacion, naming, specs base y orquestacion antes de una fase de hardening o de UX.

## Checklist

1. Confirmar stack real vigente.
2. Corregir README y docs raiz.
3. Verificar nombre real del paquete.
4. Actualizar `sdd/core` y la feature spec correspondiente.
5. Rebaselinar `.antigravity/team/`.
6. Cerrar con `lint`, `build`, commit y push.
