# Anclora Landing Page Conversion

Construir landings que se sientan como un sistema editorial premium, no como una SaaS genérica.

## Flujo
1. Leer `DESIGN.md` antes de diseñar cualquier landing.
2. Identificar una única conversión principal y como mucho una secundaria.
3. Resolver la página en 5 bloques máximos:
   - hero
   - proof strip
   - capability narrative
   - workflow o system block
   - final CTA
4. Priorizar una promesa fuerte por pantalla.
5. Mantener el producto como protagonista, sin ruido ni taxonomías excesivas.

## Reglas visuales
- Usar fondos cálidos y oscuros, nunca azulados.
- Usar el dorado ámbar del sistema como acento principal.
- Evitar layouts plantilla con grids repetitivos y simétricos por defecto.
- Favorecer titulares grandes, masas oscuras, silencios visuales y profundidad tonal.
- Mantener bordes en modo fantasma; si un borde llama la atención, sobra.
- Usar el logo como recurso atmosférico ampliado y translúcido cuando ayude a la composición.

## Reglas de conversión
- CTA principal único y visible desde el hero.
- CTA secundario solo si reduce fricción y no compite.
- No introducir múltiples caminos de navegación si no ayudan a convertir.
- Cada sección debe reforzar autoridad, claridad o deseo de entrar al producto.

## Reglas de copy
- Escribir con tono sobrio, ejecutivo y preciso.
- Evitar hype, frases vacías y clichés de IA.
- Presentar capacidades como ventajas estratégicas, no como listas de features.

## Reglas de implementación
- Respetar el lenguaje visual existente del repo si ya hay tokens y componentes útiles.
- Preferir reutilizar `Button`, `Badge`, `SurfaceCard` y el sistema tipográfico existente.
- Si hace falta estilo específico de landing, añadirlo con intención y sin contaminar el dashboard.
- Mantener la landing rápida, escaneable y mobile-first.

## Validación
- Comprobar `lint` y `build`.
- Revisar que el hero sea legible y potente en desktop y móvil.
- Revisar que el CTA principal sea visible sin esfuerzo.
- Revisar que la página no parezca una landing SaaS intercambiable.
